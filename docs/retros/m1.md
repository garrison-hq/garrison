# M1 retro — event bus and supervisor core

**Shipped:** 2026-04-22, branch `001-m1-event-bus`
**Ship gate:** all 10 acceptance steps pass against `make docker` + fresh `postgres:17` (see [specs/m1-event-bus/acceptance-evidence.md](../../specs/m1-event-bus/acceptance-evidence.md))

## What shipped

- `supervisor` Go binary (Go 1.25, `CGO_ENABLED=0`, 18 MB static) exposing `--version`, `--migrate`, and a normal run mode.
- Runtime image `garrison/supervisor:dev` built from alpine:3.20 (24 MB) via a 10-line Dockerfile.
- Postgres schema under `migrations/` (2 goose migrations): `departments`, `tickets`, `event_outbox`, `agent_instances`, the two partial indexes verbatim from `m1-context.md`, and the `ticket_created_emit` trigger + `emit_ticket_created` function.
- sqlc-generated typed query layer in `supervisor/internal/store/`.
- `internal/pgdb`: pool + dedicated LISTEN connection with the FR-017 100ms→30s backoff dialer, plus `AcquireAdvisoryLock` wrapping the fixed FR-018 key.
- `internal/events`: handler registry (`Dispatcher`), LISTEN loop, fallback poll, and reconnect state machine (connect → advisory lock → one-shot recovery → poll → LISTEN; reconnect variant skips recovery).
- `internal/spawn`: per-event dedupe transaction, concurrency-cap check, `InsertRunningInstance`, `exec.CommandContext` with per-stream line-scanning goroutines, SIGTERM→5s→SIGKILL shutdown, and the single terminal transaction.
- `internal/recovery.RunOnce` — one-shot reconcile of `status='running'` rows older than the NFR-006 5-minute window.
- `internal/concurrency.CheckCap` — two-query cap check with cap=0 as the FR-003 pause signal.
- `internal/health` — `/health` HTTP server, 200 iff 500ms `SELECT 1` ping succeeds AND `time.Since(LastPollAt) ≤ 2·PollInterval`, else 503.
- `cmd/supervisor` wires everything under `errgroup.WithContext`, SIGTERM/SIGINT/SIGHUP → root ctx cancel, exit codes per contract.
- Tests: ~35 unit tests (pkg-local), a full `integration` tag suite covering US1–US3, recovery, `/health`, cap enforcement, advisory-lock contention, and a 100-ticket volume test, plus a `chaos` tag suite for reconnect, external SIGKILL, and graceful-shutdown-with-inflight.

## What the spec got wrong (and what we learned from running it)

1. **LISTEN/poll in-flight race was not in the spec.** FR-006's dedupe transaction (`LockEventForProcessing` + `MarkEventProcessed`) only catches events that have already been *completed*. Between the dedupe-tx commit (row locked, handler running) and the terminal-tx commit (`processed_at` set), `processed_at` is still NULL — so a fallback poll fired during that window selects the event and tries to spawn a second subprocess. The 100-ticket volume test caught this cleanly (only 50 of 100 succeeded on the first run). Fix: a `sync.Map` on the `Dispatcher` keyed by `event_id`, `LoadOrStore`-gated, `defer Delete` on handler return. Sound under FR-018's single-supervisor invariant; would need revisiting if we ever relaxed that.
2. **Shutdown-path terminal writes used the cancelled root context.** Plan step 7 says "terminal tx writes `UpdateInstanceTerminal` + `MarkEventProcessed`", but didn't call out that during graceful shutdown the root ctx is already `Done()` by the time the terminal tx runs. Result: the terminal write failed with `context.Canceled`, leaving `agent_instances.status='running'`. Fix: introduce `TerminalWriteGrace = 5s` and use `context.WithTimeout(context.WithoutCancel(ctx), TerminalWriteGrace)`. The chaos test for graceful shutdown found this.
3. **Reconnect-path nil-conn deref was unspecified.** Plan's reconnect pseudo-code says "backoff → dial → poll → LISTEN" but didn't explicitly structure the dial retry. The initial implementation fell through to `listen(nil, …)` on a failed dial, panicking. Fix: inner `for conn == nil { … }` retry loop in `reconnect.go`. The chaos test for reconnect exercised the drop+redial path and found this.
4. **`agent_instances.pid` is never written.** `InsertRunningInstance` happens before `cmd.Start()` (the dedupe-tx must commit before we block on exec), so the pid isn't known at insert time, and `UpdateInstanceTerminal` doesn't `SET pid`. The pid is still emitted on every subprocess log line (`"pid": 15`) so observability is fine, but the column is dead. Not blocking for M1 acceptance; the cleanest fix is a second `UPDATE` right after `cmd.Start()`. Deferred to M2.
5. **Runtime base image was wrong for M1's fake agent.** The T001 Dockerfile used `gcr.io/distroless/static-debian12`, which has no `/bin/sh`. Since `ORG_OS_FAKE_AGENT_CMD` is `sh -c "…"` for M1, the supervisor couldn't exec the placeholder agent against the release image. Fixed in T017 by switching to `alpine:3.20` (+ca-certificates); +7 MB over distroless, acceptable for M1. M2 replaces the fake agent with the real `claude` CLI, at which point the runtime base should be revisited.
6. **Acceptance step 9 is under-specified after a clean shutdown.** "Restart the supervisor; the recovery query marks any leftover running instances as failed" has nothing to reconcile immediately after step 8's clean SIGTERM — the shutdown path writes terminal rows, so no row is stale. T017 interpreted the step as "demonstrate recovery actually works" and injected a synthetic 10-min-stale row via direct SQL before restart. Reasonable; the acceptance doc should be amended in M2 prep to clarify the expected path (either "inject a stale row" or "kill -9 the supervisor before restart").

## Dependencies added outside the locked list

Exactly one direct dep outside `AGENTS.md` §"Supervisor (Go) — Locked dependency list":

- **`github.com/google/shlex`** (`v0.0.0-20191202100458-…`) — POSIX-like argv splitter for `ORG_OS_FAKE_AGENT_CMD`. The `m1-context.md` reference command is `sh -c 'echo hello from $TICKET_ID; sleep 2'`, which a naive `strings.Fields` split mangles (splits inside the quoted argument). Alternatives considered: (a) implement a minimal shell-splitter in-tree (~60 lines; we'd still have to get quoting and escapes right, which is exactly what shlex already solves correctly); (b) require operators to pre-tokenize via a JSON array env var (awkward and breaks the m1-context example). Shlex is a single-file stdlib-only package with no transitive deps. Accepting it.

All other deps in `go.mod` are either locked (pgx/v5, goose/v3, testcontainers-go, testify, errgroup) or indirect pulls from those.

## Plan-level open questions — resolved

Both explicitly required by T018.

1. **`github.com/google/shlex` — accept or implement minimal whitespace-split?** → **Accepted.** Justified above; commit message on T008 carries the per-AGENTS.md soft-rule justification.
2. **SIGHUP semantics — separate handling or "treat as SIGTERM"?** → **Treated as SIGTERM.** `cmd/supervisor/signals.go` registers SIGTERM, SIGINT, SIGHUP together in a single `signal.Notify`; any of them cancels the root context and drives the same graceful-shutdown path. Comment in `signals.go` cites this retro. The rationale: container runtimes commonly send SIGHUP on restart, and "reload config without stopping" isn't a feature M1 needs — config is env-driven and immutable per process.

## Open questions deferred to M2

- **Per-agent-type concurrency sub-caps.** RATIONALE §11 already defers this past M1; flagged again here because the 100-ticket volume test made clear the per-department cap is what governs wall-clock throughput, and M2's CEO agent will want differentiated budgets.
- **`/metrics` endpoint.** Deferred in context; re-raise when MemPalace + dashboard land.
- **`agent_instances.pid` backfill.** See §4 above.
- **Blue/green / rolling deploys.** Disallowed in M1 by FR-018 (advisory-lock serialisation). If M2+ wants overlap, we need a per-supervisor-instance key rather than the fixed M1 key.
- **Operator-tunable fallback poll batch size.** NFR-009 fixed it at 100 for M1; revisit if volume tests ever exceed it.
- **Real `claude` CLI integration in the lifecycle manager.** Subprocess contract, auth, rate limits, cost accounting — all M2. Worth a dedicated design doc before touching `internal/spawn` again.
