M7.1 closes the gap M7 deliberately left open: ticket agents now execute
inside their per-agent sandboxed containers instead of as supervisor
children. `UseDirectExec` defaults to false; setting it true restores the
legacy path as the soak-window rollback lever. Implementation ran as a
sequential multi-agent workflow (one implement agent + one adversarial
verify agent per task, halt-on-failure); 17 of 18 tasks passed verification
with zero fix rounds, one (T016) took a single fix round.

## What the spec/plan got wrong

1. **Migration version collision.** The plan named
   `20260610000000_m7_1_agent_md_container_wording.sql`; that version number
   was already taken by `20260610000000_agents_changed_trigger.sql` and goose
   hard-errors on duplicates. Shipped as `20260610000001_…`. Lesson: check
   the migrations directory for same-day versions at plan time.
2. **`access_log stdio:/dev/stdout` crash-loops the pinned squid.** Plan §10's
   literal config FATALs after squid's privilege drop (the proxy user cannot
   reopen the root-owned stdout pipe). Shipped: log to
   `/var/log/squid/access.log`, which the image entrypoint `tail -F`s to
   stdout — FR-009 observability via `docker logs` preserved. The spike ran
   squid but never exercised this logging shape.
3. **Identical-path binds need host-side ownership prep.** Supervisor-side
   `MkdirAll` lands on the host (that's the point), but the host trees must
   be owned by the supervisor uid (100:101) or spawns fail. Cost one
   `spawn_failed` on the dev box; now an ops-checklist step.
4. **The default flip broke the supervisor-binary test harness.** T012 routed
   the M2.1/M6 binary-boot tests onto the container path (impossible without
   a socket proxy). The plan's "legacy default restored in tests that
   construct config from env" intent was right but didn't name the harness;
   `test_helpers_test.go` now pins `GARRISON_USE_DIRECT_EXEC=true`.
5. **Adjudicate is sealed, so 125–127 mapping needed a seam.** The plan
   assigned the 125–127→`spawn_failed` mapping to the transport without
   noticing `Adjudicate` (sealed surface) has no spawn-failure row; T011
   implemented it as a per-spawn terminal-reason override. Right outcome,
   plan under-specified the mechanism.
6. **Chaos blackhole ≠ dropped packets on an internal network.** The agents
   network has no default route, so off-subnet IPs fail fast with
   ENETUNREACH instead of hanging. The real blackhole stand-in is a
   resolvable host with a closed port (claude retries past any budget,
   exactly spike F3). The chaos test encodes this.
7. **Pre-spec catch that deserves the record**: `/speckit.analyze` C1 — the
   plan's reconcile predicate (`last_grandfathered_at IS NOT NULL`) would
   have stranded hired agents out of shape convergence forever. Fixed before
   implementation; T014 pins it with the 409-colliding hired-agent case.

## Gotchas worth remembering for the next milestone

1. **pid is NULL for container-executed instances** — by design (the exec's
   host-namespace PID is not a supervisor child). Anything that reads
   `agent_instances.pid` must treat NULL as "container path", not "missing".
2. **The shape hash covers the whole create body** — any intentional shape
   change (image bump, cap change, mount change) recreates every container
   at next boot. That's the design; expect a `removed`+`created` event pair
   per agent and don't read it as flapping.
3. **Rollback mode is one env var** (`GARRISON_USE_DIRECT_EXEC=true`) and
   was live-verified to produce zero agent-container API calls. Direct-exec
   agents still get mempalace MCP; container agents don't (Q1) — agent_md
   wording is mode-neutral.
4. **Secrets transit exec-create Env only** — container `docker inspect`
   shows no Env at all now. The vaultlog `UnsafeBytes` call-site count is
   still two; `appendSecretEnv` is the shared injection seam.
5. **Integration suites need `-p 1` on docker-constrained hosts** — address
   pools, not flakiness.
6. **M9 scheduled wake-ups land on this substrate**: the proactive-spawn
   path should reuse `runRealClaudeViaContainer` + the per-agent inflight
   slot untouched; only the trigger differs.
