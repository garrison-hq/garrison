# M4 tasks — operator dashboard mutations

**Branch**: `009-m4-dashboard-mutations` | **Plan**: [plan.md](./plan.md) | **Spec**: [spec.md](./spec.md) | **Context**: [m4-context.md](../_context/m4-context.md)

22 tasks total, executed linearly by a solo operator. Each task is one Claude Code session in scope and produces a reviewable commit. The repo is in a working state after every task.

---

## Ordering principle

M4 is the first mutation-bearing milestone. The dependency chain is: cross-boundary schema → Drizzle re-introspection → mutation primitives (audit / locks / UI) → vault subsystem (the heaviest surface) → ticket mutations → agent settings + supervisor cache invalidator → suspected-secret pattern category (supervisor scanner + hygiene UI) → standalone harness migration → cross-cutting validation → deployment → acceptance → retro.

The two supervisor-side changes (T014 cache invalidator, T015 scanner extension) ship within the milestone but are narrowly scoped — each is a focused edit to one supervisor package. They land after the dashboard side that consumes / produces the corresponding signals.

The Infisical TS SDK (`@infisical/sdk`) is the one principled new direct dep on the dashboard side; justified in the T006 commit message and recapped in the M4 retro per AGENTS.md soft-rule.
