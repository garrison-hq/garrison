Strict dependency order. M2.3 extends the shipped M2.2.x supervisor; it does not rebuild anything. After every task the repository is in a working state — the supervisor binary compiles, all M1 + M2.1 + M2.2 + M2.2.1 + M2.2.2 tests still pass, and the newly-added behaviour is independently verifiable by the completion condition of that task.
