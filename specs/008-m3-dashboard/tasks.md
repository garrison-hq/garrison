**Branch**: `008-m3-dashboard` | **Plan**: [plan.md](./plan.md) | **Spec**: [spec.md](./spec.md) | **Context**: [m3-context.md](../_context/m3-context.md)

23 tasks total, executed linearly by a solo operator. Each task is one Claude Code session in scope and produces a reviewable commit. The repo is in a working state after every task.

---

## Ordering principle

M3 is the first frontend-bearing milestone, so T001–T002 are genuinely greenfield (one cross-boundary goose migration + one dashboard scaffolding task). The rest extends what T002 establishes.

The dependency chain inside the dashboard subsystem is: schema → DB clients → auth → shell+i18n → read-side surfaces → live activity feed → cross-cutting validation → deployment → acceptance → retro. Each phase fully completes before the next begins; tasks within a phase have explicit `Depends on` lines.

Dashboard-side deps (Next.js 16, React 19, Drizzle + drizzle-kit, better-auth, postgres-js, next-intl, TanStack Query, Tailwind v4, Vitest, Playwright, `@tanstack/react-virtual`) all land in M3. The supervisor's locked-deps streak is supervisor-scoped and unaffected; each dashboard dep is justified in its task's commit message and recapped in the retro per AGENTS.md soft-rule.
