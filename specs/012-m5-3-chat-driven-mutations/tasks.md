## Ordering principle

M5.3 extends a shipped M5.1 + M5.2 substrate. The threat-model amendment (T001) is load-bearing per FR-400 and lands BEFORE any `garrison-mutate` verb code commits — same M2.3 vault-threat-model-first pattern. The sealed MCP allow-list test (T003) passes BEFORE the verb scaffolding (T004) per FR-433, so any verb-code drift that would silently leak a fourth MCP server entry fails CI immediately.

Verbs land in dependency order: ticket verbs first (T005–T007) since `create_ticket` is the canonical use case from `ARCHITECTURE.md:425` and the integration test path; agent verbs next (T008–T010) including the leak-scan-coupled `edit_agent_config`; hiring verb last (T011) since it depends on the new `hiring_proposals` table from T002 but doesn't depend on any other verb.

Supervisor-side chat substrate extension (T012) lands once the verbs work, so the per-turn tool-call ceiling can be exercised against real verb invocations. Dashboard SSE union extension (T013) follows the supervisor's transport extension. Chip component (T014) and stream integration (T015) close the observability layer. Activity feed (T016) and hiring stopgap (T017) round out the cross-cutting surfaces.

Architecture amendment (T018) lands once the surface compiles end-to-end. Mockclaude fixtures (T019) and chaos tests (T020) come before Playwright (T021) so the integration tests have stable fixtures to drive against. Acceptance (T022) is the scripted SC walk; retro (T023) is the last task.

Zero new dependencies in either `supervisor/go.mod` or `dashboard/package.json` per FR-540. The locked-deps streak preserved.

Task IDs use three-digit zero-padded form (`T001` … `T023`) matching the M5.1 / M5.2 milestone precedent.
