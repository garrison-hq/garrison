## Ordering principle

M5.4 extends a shipped M5.3 substrate. The architecture amendment (T002) lands SECOND so the FR-501 substring-pinned amendment test passes BEFORE any subsequent task could quietly ship code without keeping the docs in sync — same pattern as M5.3's T018 / T012 ARCHITECTURE-amendment-first discipline.

Migrations + sqlc query (T001) come first because the supervisor's `dashboardapi` auth middleware (T008) depends on the new `GetBetterAuthSessionByID` query and the `GRANT SELECT ON better_auth_session TO garrison_supervisor` permission. The amendment-first test (T002) follows immediately so the rest of the milestone can't merge without keeping ARCHITECTURE.md in sync.

Supervisor-side packages land in dependency order: config (T003) → objstore skeleton (T004) → objstore MinIO calls (T005) → objstore integration test (T006) → mempalace QueryClient (T007) → dashboardapi skeleton + auth (T008) → dashboardapi handlers (T009, T010) → cmd/supervisor lifecycle wiring (T011). Each phase passes its own tests before the next builds on it.

Compose / docker (T012) lands once the supervisor binary fully wires the new components, so the post-T012 stack can boot end-to-end against real MinIO.

Dashboard side follows the same data-layer-then-components shape M5.2 + M5.3 used: actions + queries first (T013, T014), then the leaf editor component (T015), then the parent tab components (T016, T017), then the orchestrating `KnowsPane` that replaces the M5.2 placeholder (T018).

Playwright (T019) lands after both the supervisor stack AND the dashboard surface compile end-to-end, so the spec exercises real round-trips through the proxy. Ops checklist (T020) follows since its content is informed by what implementation actually produced.

Acceptance (T021) is the scripted SC walk; retro (T022) is the last task — the M5 arc closes here.

Two new locked-deps additions: `github.com/minio/minio-go/v7` (Go) lands in T004 when first needed; `@uiw/react-codemirror` + `@codemirror/lang-markdown` (TS) land in T015. Both flagged in the M5.4 retro per AGENTS.md.

Task IDs use three-digit zero-padded form (`T001` … `T022`) matching the M5.1 / M5.2 / M5.3 milestone precedent.
