## Ordering principle

M5.1 introduces a new supervisor subsystem (`internal/chat/`) that depends on three cross-cutting refactors landing first: schema (chat_sessions + chat_messages), parser-policy abstraction (`spawn.Policy`), and parser event surface (`claudeproto.OnStreamEvent`). After foundations, the chat package builds bottom-up — types → persistence → transcript / transport → policy → spawn → worker → listener. Then build artefacts (Docker images), wiring (compose + main.go), then dashboard surfaces, then integration / chaos / Playwright tests, then the scripted acceptance check, then retro.

Zero new Go dependencies. Zero new TS dependencies. The chat container is a runtime artefact built out-of-band via `make chat-image`; the supervisor binary stays Go-only per Constitution VII and operator's confirmation on Q-C.

The `internal/dockerexec/` rename (T002) is folded into the milestone per operator's Q-D answer; both mempalace and chat depend on it.
