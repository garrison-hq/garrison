- [ ] T001 Add `internal/finalize/position.go` decode-position helper and unit tests
  - **Depends on**: M2.2.1 shipped
  - **Files**: [supervisor/internal/finalize/position.go](../../supervisor/internal/finalize/position.go) (new), [supervisor/internal/finalize/position_test.go](../../supervisor/internal/finalize/position_test.go) (new)
  - **Completion condition**: new private function `decodePosition(payload []byte, offset int64) (line, column int, excerpt string)` per [plan.md](./plan.md) §"`internal/finalize/position.go`". Pure stdlib; no new dependencies; ~30–40 LOC. Behavioural properties: (a) empty payload OR `offset <= 0` → `(1, 1, "")`; (b) `offset > len(payload)` → line/column at EOF position + last 40 chars (or whole payload if shorter); (c) in-bounds offset → 1-based line walked from start, column relative to start of current line, 40-char excerpt centred on offset (±20 chars, clamped to payload bounds); (d) excerpt sanitises ASCII control characters to `·` so the value is safe to embed in a `hint` string and a log line. Four unit tests pass: `TestDecodePositionAtOffsetZero` (empty payload + offset=0 → `(1,1,"")`), `TestDecodePositionMidStream` (offset in middle of multi-line JSON → correct line/column + 40-char excerpt centred on offset), `TestDecodePositionPastEOF` (offset > len(payload) → EOF line/column + last-40-chars excerpt), `TestDecodePositionMultilineNewlines` (offset on the 3rd line of a 5-line payload → line=3, column counted from start of line 3). `go test ./internal/finalize/... -count=1` passes; M2.2.1 finalize tests unchanged.
  - **Out of scope for this task**: wiring the helper into `Validate()` ([T002](#t002)); any change to existing `tool.go` types; any change to `handler.go`; sanitisation of non-ASCII / multi-byte UTF-8 (the helper treats the payload as bytes per `*json.SyntaxError.Offset`'s contract; multi-byte sequences are sliced byte-wise — implementer should not "be helpful" by adding rune-aware logic without a concrete failing case).
